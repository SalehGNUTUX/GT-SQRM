## 📖 نبذة عن البرنامج

**GT-SQRM** هو تطوير سطح المكتب للتطبيق الشهير **GT-SQR** (نسخة المتصفح). تم بناء هذا الإصدار خصيصاً لنظام GNU/Linux ليمنحك حرية كاملة في إنشاء مقاطع فيديو قصيرة (ريلز) للقرآن الكريم بجودة احترافية، دون القيود التي تفرضها المتصفحات.

سواء كنت صانع محتوى، داعية، أو خطيب، يتيح لك GT-SQRM إنشاء فيديوهات قرآنية مبهرة مع تلاوات، خلفيات متحركة، تأثيرات بصرية، وشعارات مخصصة — بكل سهولة.

## ✨ المميزات الرئيسية

### 🎬 التصدير بجودة احترافية
- **محرك ffmpeg مدمج** – تصدير الفيديو بأعلى جودة ممكنة (بدون سقف).
- **صيغ متعددة**: MP4 (H.264/H.265) · WebM (VP9) · MKV (AV1).
- **تحكم كامل بجودة CRF** – كلما قلت القيمة، كانت الجودة أفضل.
- **إمكانية التصدير في الخلفية** دون توقف.

### 🖼️ خلفيات متنوعة
- **تدرجات لونية** قابلة للتخصيص بالكامل.
- **صور ثابتة** من جهازك.
- **فيديوهات خلفية** مع إمكانية التحكم بالحركة (زحف، تكبير، تحريك).
- **استيراد مباشر من يوتيوب** باستخدام yt-dlp – فيديوهات وصوتيات.

### 🕌 إعدادات متقدمة للقرآن
- **أكثر من 30 قارئ** مشهور (مع إمكانية إضافة قراء جدد).
- **الترجمة والتفسير** – دعم التفاسير العربية والترجمات العالمية.
- **خطوط قرآنية متعددة** – خطوط مدمجة، محلية، وخطوط النظام.

### ✨ التأثيرات البصرية
- **فلترات لونية**: طبيعي، أبيض وأسود، دافئ، بارد.
- **زخارف إسلامية**: سداسية، هندسية، نجوم، أقواس، إطارات.
- **تأثيرات متقدمة**: توهج، نيون، ظل ثلاثي الأبعاد، Pixelate، Mosaic، Ripple، Wave، Swirl، Kaleidoscope، Glitch، Old Film.
- **إضاءة متغيرة**: تحكم بالتعتيم، السطوع، والتشبع.

### 🔊 نظام صوتي متطور
- **فصل تام بين صوت المعاينة والتصدير** – يمكنك كتم المعاينة دون التأثير على الفيديو الناتج.
- **صوت خلفية** مع إمكانية رفع ملفات MP3، OGG، FLAC، WAV.
- **موجات صوتية متحركة** مع 6 أشكال مختلفة (أعمدة، موجة، نقاط، مرآة، دائرة، طيف).

### 🎯 ميزات إضافية
- **قوالب قابلة للحفظ** – احفظ إعداداتك المفضلة واسترجعها لاحقاً.
- **تصدير دفعات (Batch)** – قم بإنشاء قائمة بمشاريع متعددة وتصديرها تباعاً.
- **علامة مائية** – أضف توقيعك على الفيديوهات.
- **شعار مخصص** – صورة أو فيديو MOV شفاف.
- **دعم كامل للغة العربية** مع واجهة سهلة الاستخدام.

## ⚠️ المشاكل الحالية (الإصدار 1.0.0)

1. **كتم الصوت** – عند كتم المعاينة (باستخدام زر 🔊 في الشريط السفلي)، **يؤثر ذلك حالياً على الملف الناتج** (أي إذا تم كتم المعاينة، سيتم تصدير الفيديو بدون صوت). هذا السلوك غير مرغوب وسيتم إصلاحه في الإصدارات القادمة.
   - **الحل المؤقت**: لا تستخدم زر كتم المعاينة إذا كنت تخطط لتصدير الفيديو بصوت. بدلاً من ذلك، يمكنك خفض مستوى الصوت إلى 0 في شريط التحكم.

2. **بدء فيديو الخلفية من البداية عند التصدير** – عند تصدير الفيديو، **لا يبدأ فيديو الخلفية من البداية دائماً**، بل قد يبدأ من النقطة التي كان فيها أثناء المعاينة.
   - **الحل المؤقت**: قبل التصدير، تأكد من إيقاف تشغيل الفيديو وإعادة تشغيله من البداية يدوياً.

3. **مجلد الحفظ الدائم** – خاصية اختيار مجلد دائم لحفظ الوسائط المحمّلة (عبر yt-dlp) **لا تعمل بشكل صحيح**، ويتم حفظ الملفات دائماً في المجلد المؤقت (/tmp).
   - **الحل المؤقت**: بعد تحميل الملف، يمكنك نسخه يدوياً من المجلد المؤقت إلى المكان الذي تريده.

4. **أيقونة التطبيق في قائمة البرامج** – عند تثبيت حزمة Deb أو تشغيل AppImage، **قد لا تظهر أيقونة التطبيق بشكل صحيح** في قائمة البرامج في بعض بيئات سطح المكتب.
   - **الحل المؤقت**: يُنصح باستخدام **GearLever** لدمج التطبيق في القائمة، أو إنشاء إدخال يدوي عبر ملف `.desktop`.

5. **تحميل الآيات في وضع عدم الاتصال** – في حالة عدم وجود اتصال بالإنترنت، **يعمل التطبيق بشكل محدود** مع سورة الفاتحة فقط كاحتياطي، ولا يمكن تحميل سور أخرى.
   - **الحل المؤقت**: تأكد من وجود اتصال إنترنت عند بدء التطبيق لتحميل قائمة السور كاملة.

## 📦 التثبيت عبر GearLever (لدمج التطبيق في قائمة البرامج)

**GearLever** هي أداة رائعة تتيح لك دمج ملفات AppImage في قائمة البرامج بسهولة. وهي متوفرة عبر مستودعات Flatpak.

### تثبيت GearLever

```bash
# من Flatpak
flatpak install flathub it.mijorus.gearlever

# أو من مستودع التوزيعة (إن وجد)
sudo apt install gearlever  # لدبيان/أوبونتو
sudo dnf install gearlever  # لفيدورا
```

### استخدام GearLever مع GT-SQRM

1. حمّل ملف `GT-SQRM-1.0.0.AppImage` من صفحة الإصدارات.
2. افتح تطبيق GearLever.
3. اسحب ملف AppImage إلى نافذة GearLever.
4. سيظهر التطبيق تلقائياً في قائمة البرامج مع الأيقونة الخاصة به.

### إنشاء إدخال يدوي (بديل)

إذا لم ترغب في استخدام GearLever، يمكنك إنشاء إدخال يدوي:

```bash
# نسخ ملف .desktop إلى المجلد المحلي
cp resources/GT-SQRM.desktop ~/.local/share/applications/

# تحديث قاعدة البيانات
update-desktop-database ~/.local/share/applications/
```

## 🛠️ للمطورين: البناء من المصدر

### المتطلبات
- Node.js 18+
- npm
- ffmpeg (اختياري للتطوير)
- yt-dlp (اختياري للتطوير)

### الخطوات

```bash
git clone https://github.com/SalehGNUTUX/GT-SQRM.git
cd GT-SQRM
npm install
npm run dev  # للتشغيل في وضع التطوير
npm run build  # لبناء AppImage
```

### هيكل المشروع

```
GT-SQRM/
├── src/
│   ├── main/
│   │   └── main.js          # Main Process — إدارة النافذة + IPC
│   ├── preload/
│   │   └── preload.js       # الجسر الآمن بين Main و Renderer
│   └── renderer/
│       ├── index.html       # الواجهة
│       ├── app.js           # منطق التطبيق
│       ├── fonts/           # الخطوط المحلية
│       └── GT-SQRM-icons/   # أيقونات التطبيق
├── resources/
│   └── bin/                 # ffmpeg + yt-dlp (تُحمَّل بواسطة build script)
├── scripts/
│   ├── build-appimage.sh    # بناء AppImage كامل
│   └── dev-setup.sh         # إعداد بيئة التطوير
└── package.json
```
